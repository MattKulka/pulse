# Pulse — Live Data Dashboard Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a fluid, animated analytics dashboard on the live USGS earthquake feed, with linked charts, brush-to-cross-filter, synchronized crosshair, and smooth D3 transitions.

**Architecture:** Vite + React 18 + TS(strict). TanStack Query fetches/caches the USGS `all_day.geojson` feed. Data is normalized to a `Quake[]`. A single Zustand store holds cross-chart UI state (brush range, hovered id, pinned id, hidden series). A memoized `filteredQuakes` selector is the one source of truth feeding KPI tiles and all charts. D3 computes geometry (scales/shapes/transitions) rendered into React-controlled SVG.

**Tech Stack:** React 18, TypeScript strict, Vite (port 5175), TanStack Query, Zustand, D3, TopoJSON, Tailwind, Vitest, Playwright.
